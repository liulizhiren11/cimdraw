# 当前状态

最后整理日期：2026-06-20

## 1. 工程定位

`cimdraw` 当前定位为面向电力一次接线图的 Qt/C++ 桌面工程，重点覆盖图元编辑、连接关系表达、拓扑分析和运行态数据接入。

仓库中与当前电力工程主线无关的内容已经逐步清理，文档也只保留当前工程真实需要维护的部分。

## 2. 当前已具备能力

1. 基于 `QGraphicsScene / QGraphicsView` 的图纸编辑框架。
2. 电力一次设备图元、工具箱和基础交互。
3. 连接点、连接关系和场景级拓扑索引。
4. 导通、供电路径、接地路径等本地拓扑分析能力。
5. 基于 Qt Property Browser 的属性查看与编辑。
6. `lz_data.json` 运行态数据源接入能力。

## 3. 当前演进方向

当前新一轮设计重点是把 `cimdraw` 从“图形编辑器 + 拓扑分析器”扩展为“面向 CIM 的电力模型工作台”，分阶段推进：

1. `P0`：打通 `EQ + SSH` 导入、对象树浏览、图纸初稿和双向定位。
2. `P1`：补齐校验、拓扑分析接入、回写和最小导出。
3. `P2`：补齐批处理、差异比较和统一外接分析接口。
4. `P3`：补齐 `DL / GL`、查询缓存和平台化治理能力。

## 5. 当前约束

1. 只保留与当前电力工程主线直接相关的代码和文档。
2. 新增设计必须优先复用现有 `src/topology`、`src/wiring/power` 和场景编辑框架。
3. `CIM` 能力优先按最小闭环推进，不在 `P0` 阶段追求全量标准覆盖。
