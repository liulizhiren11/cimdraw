# CIM 标准图元缺口与补齐设计

最后整理日期：2026-06-29

## 1. 目标

本文件用于明确 `lz-power` 在适配标准 CIM/CGMES 过程中，当前图元层还缺哪些标准一次设备表达、这些缺口会造成什么问题、后续应按什么优先级补齐。

本文件只讨论“该不该有对应图元表达”和“如何补齐图元能力”，不讨论完整导入布置算法。导入拓扑分组、正交布线和布局优先级方案单独放在《CIM拓扑分组与正交布置设计.md》。

如果要直接看“当前还能做什么、下一轮具体排什么”，优先配合《CIM标准图元补齐执行ToDoList.md》一起使用。

标准参考入口：

1. UCAIug CIM 模型概览：
   https://cim-mg.ucaiug.io/section4-cim-overview/?utm_source=chatgpt.com

## 2. 当前现状

当前导入链路已经具备下面几项基础：

1. 可以导入 `EQ / SSH / TP / SV / DL / GL` 等 CGMES 文件。
2. 可以把对象装入 `CimModel`，并在对象浏览器中查看。
3. 已具备一批核心图元映射，已覆盖母线、开关、变压器、负荷、电源、导体、补偿、接地、避雷器以及部分测量设备。
4. `Terminal / ConnectivityNode / TopologicalNode` 已进入模型，但主要用于语义关系和连线推断，不应直接当作一次设备图元绘制。
5. 对象浏览器已支持按“已可视化 / 应显示未实例化 / 语义对象 / 布局对象 / 元数据容器对象 / 抽象层对象 / 未映射对象”标记对象状态。
6. 导入摘要已支持显示“应可视化设备数 / 已可视化设备数 / 未可视化设备数 / 可视化覆盖率 / 非图元对象数”。

当前主要问题不是“对象没导入”，而是“标准设备对象没有足够的图元承接”，导致：

1. 大量对象只能存在于语义层，不能在图纸上正确表达。
2. 导入后的图纸只能展示局部设备，不能形成可读的一次接线图。
3. 连线即便能建立，也缺少与设备类型对应的标准视觉语义。
4. 后续做对象定位、拓扑高亮、回写和分析结果着色时，图形层承接不足。

## 3. 图元补齐原则

### 3.1 语义对象与图元职责分离

1. `Equipment / ConductingEquipment` 及其具体子类，是图元映射的主要来源。
2. `Terminal` 是端子语义对象，应映射为图元连接点，不应单独画成一次设备。
3. `ConnectivityNode / TopologicalNode` 是连接和拓扑分组核心，不应默认画成一次设备。
4. `ACLineSegment`、母线连接关系、间隔关系应优先转成线路或导体表达，而不是普通矩形节点。

### 3.2 标准优先

1. 优先补齐标准 CIM/CGMES 中一次接线图最常见、最影响可读性的设备。
2. 优先补齐当前工程已有图元能力附近的设备，减少第一轮实现成本。
3. 对暂时没有图元的标准对象，允许先导入语义模型，但必须在导入报告中说明“已导入、未可视化”。

### 3.3 分层落地

1. 第一层：补齐图元类型枚举和 `CimGraphicMapper` 映射。
2. 第二层：补齐图元连接点规则、默认朝向和默认尺寸。
3. 第三层：补齐运行态、拓扑角色、属性显示和结果着色。
4. 第四层：补齐测试样例、自动化测试和验收清单。

2. 待细化：已有基础承接，但标准表达还需要继续细分。
3. 仍缺失：当前还没有稳定落地策略。

## 5. 现有图元需要补强的点

即便已经有映射的设备，也还需要补齐下面这些能力，否则仍然不能稳定承接标准 CIM：

### 5.1 母线

1. `BusbarSection` 需要稳定的沿边连接点生成策略。
2. 需要支持多端接入时的视觉分布，避免所有连接线堆在一个锚点。
3. 需要支持按电压等级、间隔或拓扑分组后的布局锚定。

### 5.2 开关类

1. `Breaker / Disconnector / LoadBreakSwitch / GroundDisconnector` 应统一抽象为开关族。
2. 要区分开关类型、默认方向和开合状态映射。
3. 要支持从 `SSH` 的状态字段回填图元开断状态。

### 5.3 变压器

1. 当前已覆盖 `自耦变 / 接地变 / 厂站变 / 双绕组 / 三绕组`。
2. 下一步重点变成端子顺序、绕组侧方向、连接点布局和真实 `DL` 样例回归。

### 5.4 负荷和电源

1. `EnergyConsumer`、`SynchronousMachine` 目前已有基础映射，但需要明确进出线方向。
2. 后续要补齐 `ExternalNetworkInjection`、`EquivalentInjection`、`AsynchronousMachine` 等近源荷类设备。

### 5.5 导体与线路

1. `ACLineSegment` 不应退化为普通几何连接线。
2. 需要区分“设备间图形连接线”和“标准导体对象图元”。
3. 导体图元应能参与对象定位、属性显示和后续拓扑分析。

## 6. 建议的图元分层

建议把标准 CIM 设备在图元层分成四组：

### 6.1 核心一次设备

1. 母线
2. 断路器
3. 刀闸
4. 接地刀
5. 负荷
6. 发电机
7. 变压器
8. 线路

### 6.2 补偿与保护设备

1. 并联补偿
2. 串联补偿
3. 避雷器
4. 熔断器
5. 接地设备

### 6.3 测量设备

1. CT
2. PT/VT
3. 其他测量或采样附属对象

### 6.4 语义辅助对象

1. `Terminal`
2. `ConnectivityNode`
3. `TopologicalNode`
4. `BaseVoltage`
5. `VoltageLevel`
6. `Bay`
7. `Substation`

这一组默认不直接画成一次设备图元，但要参与：

1. 分组
2. 连接
3. 层级组织
4. 布局锚定
5. 对象浏览和定位

## 6.5 浏览器里常见但不应直接画成一次图元的对象

这组对象在模型浏览器里“能看见”，不代表它们应在主接线图里各自生成独立一次设备图元。用户看到“还有很多对象没显示”时，首先要按这组口径判断：

| CIM 类 | 浏览器状态 | 说明 |
| --- | --- | --- |
| `Terminal` | 语义对象 | 端子语义，服务于设备与结点连接，不单独画成一次设备 |
| `ConnectivityNode` | 语义对象 | 连通结点语义对象，不直接画成一次设备 |
| `TopologicalNode` | 语义对象 | 拓扑分组结点，不直接画成一次设备 |
| `DiagramObject` / `DiagramObjectPoint` | 布局对象 | 图形定位与折点信息，不是一次设备 |
| `Diagram` / `CoordinateSystem` | 布局对象 | 图纸与坐标系统元信息，不是一次设备 |
| `Substation` / `VoltageLevel` / `Bay` / `SubGeographicalRegion` | 元数据/容器对象 | 层级组织与归属容器，不是一次设备 |
| `BaseVoltage` / `OperationalLimit*` / `VoltageLimit` / `CurrentLimit` | 元数据/容器对象 | 参数、限值与运行约束，不是一次设备 |

因此，“浏览器对象数”和“画布图元数”本来就不会一一相等；系统需要做的是把这类差异解释清楚，而不是把所有语义对象都硬画出来。

## 7. 与标准 CIM 的适配口径

### 7.1 必须适配的对象

满足下面任一条件的标准对象，应优先适配图元表达：

1. 直接参与一次接线拓扑。
2. 在样例数据中高频出现。
3. 没有它就无法看懂图纸走向。
4. 没有它就无法完成导通、供电、接地等分析展示。

### 7.2 可先只导入不显示的对象

满足下面条件的对象，可以先只保留在语义层：

1. 主要用于元数据描述。
2. 主要用于版本、Profile、来源说明。
3. 主要用于校验、统计或布局辅助。
4. 当前没有稳定图元表达，且不影响主拓扑可读性。

### 7.3 不允许的退化策略

1. 不能把所有导电设备都降级成同一种通用图元。
2. 不能把 `Terminal` 当作设备节点画出来冒充设备实例化。
3. 不能用一条普通连接线替代本应单独存在的标准导体类对象。
4. 不能因为当前项目暂时没有图元，就把标准对象直接判错为“非法 CIM”。

## 9. 验收标准

### 9.1 功能验收

1. 导入标准样例后，核心一次设备能被正确实例化。
2. `Terminal / ConnectivityNode / TopologicalNode` 不会被误画成一次设备。
3. 对象浏览器中的已导入对象，与画布上的可视化对象范围差异可解释。
4. 未可视化的标准对象，会在导入结果中给出清晰提示。

### 9.2 测试验收

1. `CimGraphicMapper` 的新增映射全部有自动化测试覆盖。
2. 至少一组中等样例能证明新增图元被实际实例化。
3. 图元实例化数量与对象统计报告可对应。

## 10. 结论

`lz-power` 适配标准 CIM，不是把所有语义对象都直接画出来，而是要把“应显示的电气设备对象”稳定地映射为标准图元，把 `Terminal / ConnectivityNode / TopologicalNode` 这类连接语义对象留在拓扑与分组层使用。

只有先补齐这批标准设备图元缺口，后面的拓扑分组、正交连线、标准布局和分析回写才能真正做成一套可读、可用、可验收的 CIM 导入能力。
