# CIM 拓扑分组与正交布置设计

最后整理日期：2026-06-20

## 1. 目标

本文件用于定义 `lz-power` 在导入标准 CIM/CGMES 后，如何基于 `TopologicalNode / ConnectivityNode` 重建拓扑分组，如何只实例化应显示的电气设备对象，如何统一生成正交连接线，以及在存在 `DL / DI` 布局信息时如何优先使用标准布局。

本文件聚焦导入后的“一次接线图生成规则”，目标不是把对象简单摆上画布，而是让用户一眼看懂设备走向和拓扑关系。

## 2. 核心设计目标

导入后的图纸应满足下面四个目标：

1. 连接分组以 `TopologicalNode / ConnectivityNode` 为核心，而不是按对象列表顺序串起来。
2. 只实例化应显示的电气设备对象，不把 `Terminal` 这类语义对象误画成设备。
3. 连接线统一走标准正交路径，避免斜穿、长跨和整排串线。
4. 有 `DL / DI` 布局信息时优先落标准布局；没有时再进入自动正交布置。

## 3. 当前问题

当前导入后的出图方式主要存在下面几个问题：

1. 设备摆放以类别分行和顺序排列为主，不是按拓扑关系展开。
2. 连线建立主要依赖 `Terminal` 关系的后补连接，容易出现整排串接。
3. 设备没有围绕母线、间隔、拓扑节点形成局部结构。
4. 即使连线引擎支持正交路径，场景生成层没有提供正确的拓扑布局输入。
5. `DL / DI` 布局信息尚未成为优先输入，标准样例里的现成图纸坐标没有充分利用。

## 4. 总体原则

### 4.1 语义优先于几何

1. 先根据 CIM 语义关系建立拓扑分组。
2. 再决定哪些设备需要画出来。
3. 最后才生成图元坐标和正交连线。

### 4.2 标准布局优先于自动布局

1. 如果 `DL / DI` 已给出稳定坐标和图纸关联，应优先使用。
2. 如果只有局部布局信息，应优先用已有布局锚点，再补自动布局。
3. 只有在缺少标准布局时，才进入自动正交布置。

### 4.3 设备表达优先于语义节点可视化

1. 图上主要画设备，不画 `Terminal`。
2. 默认不把 `ConnectivityNode / TopologicalNode` 画成普通节点。
3. 这些对象主要用作分组、路由和连线关系推导。

### 4.4 可读性优先于对象顺序

1. 相邻设备的空间关系应反映一次接线走向。
2. 母线应成为局部结构的锚点。
3. 开关、导体、负荷、电源应沿拓扑方向展开。

## 5. 拓扑分组模型

### 5.1 分组核心

导入后先构建下面三层关系：

1. `ConductingEquipment -> Terminal`
2. `Terminal -> ConnectivityNode`
3. `ConnectivityNode -> TopologicalNode`

如果样例只有其中一层，也允许降级：

1. 有 `TopologicalNode` 时，以 `TopologicalNode` 作为主分组键。
2. 没有 `TopologicalNode` 但有 `ConnectivityNode` 时，以 `ConnectivityNode` 作为主分组键。
3. 两者都缺失时，不报标准错误；导入保留对象，并进入最小可用自动布置降级策略。

### 5.2 拓扑分组输出

建议生成中间结构 `CimTopologyLayoutModel`，至少包含：

1. 节点分组：一个节点下有哪些设备
2. 设备端子表：设备有哪些端子、端子连到哪个节点
3. 设备邻接图：设备与设备通过什么节点或导体相连
4. 母线锚点集合：哪些设备应视为母线锚点
5. 导体对象集合：哪些对象应视为独立导体或线路

### 5.3 分组规则

1. 同一 `TopologicalNode` 下的设备，应被识别为同一导通组。
2. 同一 `ConnectivityNode` 下的设备，应被识别为同一连接组。
3. `BusbarSection` 优先作为本组视觉锚点。
4. 如果组内有多个母线对象，应进一步按 `VoltageLevel / Bay / Container` 切分。
5. 如果组内没有母线，则按开关链、导体链和源荷端设备构造局部主干。

## 6. 图元实例化规则

### 6.1 应实例化的对象

默认应实例化的对象包括：

1. `BusbarSection`
2. `Breaker`
3. `Disconnector`
4. `LoadBreakSwitch`
5. `GroundDisconnector`
6. `PowerTransformer`
7. `EnergyConsumer`
8. `SynchronousMachine`
9. `ACLineSegment`
10. 其他已明确映射的标准设备类

### 6.2 不应直接实例化为设备的对象

1. `Terminal`
2. `ConnectivityNode`
3. `TopologicalNode`
4. `BaseVoltage`
5. `VoltageLevel`
6. `Bay`
7. `Substation`
8. `md:FullModel`

这些对象应进入：

1. 对象浏览
2. 拓扑分组
3. 布局锚定
4. 属性查询
5. 导入报告

### 6.3 可选辅助可视化

后续可考虑提供调试模式，让 `ConnectivityNode / TopologicalNode` 以辅助节点方式显示，但必须满足：

1. 默认关闭
2. 不冒充一次设备
3. 不污染正式一次接线图视图

## 7. 正交布线规则

### 7.1 基本要求

1. 所有自动生成的设备间连接线默认使用正交路径。
2. 路径应优先选择最少穿越、较少拐点、局部最短的方案。
3. 不允许在无必要的情况下出现斜线。
4. 不允许把一整组设备用一条超长水平线粗暴串联。

### 7.2 连接生成顺序

建议按下面顺序生成连接：

1. 母线到本组开关
2. 开关到导体或中间设备
3. 中间设备到负荷/电源
4. 跨组导体连接
5. 变压器跨侧连接

### 7.3 方向规则

1. 母线优先作为局部起始锚点。
2. 负荷和电源默认布置在母线外侧。
3. 开关应处于母线与源荷之间。
4. 变压器作为跨层设备，应位于两个母线层或两个电压层之间。

### 7.4 端口规则

1. 每类设备应定义默认进线端和出线端。
2. 母线应支持按连接目标动态分配接入点。
3. 开关设备应优先使用上下或左右成对端口，避免折线反复回绕。

## 8. 布局优先级规则

### 8.1 第一优先级：标准布局

当数据集中存在 `DL / DI` 信息时，优先读取：

1. 设备图元坐标
2. 图纸归属关系
3. 设备与图形符号的对应
4. 已有折线或连接表达

此时目标不是“重新猜一个布局”，而是“尽量还原标准数据集原有图纸”。

### 8.2 第二优先级：半自动补齐布局

当 `DL / DI` 只覆盖部分对象时：

1. 保持已有标准对象位置不动
2. 围绕已有锚点补充缺失设备
3. 新补充连线继续使用正交路由

### 8.3 第三优先级：自动正交布置

在没有 `DL / DI` 时，执行自动布局：

1. 先按 `Substation / VoltageLevel / Bay` 做外层分区
2. 再按 `TopologicalNode / ConnectivityNode` 做局部分组
3. 母线横向或纵向作为主干
4. 开关围绕母线展开
5. 负荷和电源布置在分支末端
6. 变压器置于不同层级之间

## 9. 自动正交布置建议

### 9.1 外层布局

1. 同一 `Substation` 的对象保持相对聚集
2. 同一 `VoltageLevel` 的对象尽量在同一水平带或垂直带
3. 同一 `Bay` 的对象形成紧凑局部单元

### 9.2 局部布局

1. 每个拓扑分组优先寻找母线
2. 有母线时，以母线为中心向外展开
3. 无母线时，以开关链或导体链构造主干
4. 末端设备如负荷、电源、接地应放在枝端

### 9.3 跨组关系

1. 跨电压层关系由变压器承接
2. 跨母线关系由线路或导体承接
3. 避免跨组线在主视区长距离横穿

## 10. 异常与降级策略

### 10.1 标准数据不完整

1. 如果缺少 `TopologicalNode`，但有 `ConnectivityNode`，仍应完成分组和出图。
2. 如果两者都缺失，但设备对象有效，则允许只导入并做最小可视化，不应直接报“非法文件”。
3. 如果只有元数据文件，如纯 `SV` `FullModel`，允许导入并跳过对象创建。

### 10.2 没有足够图元映射

1. 已导入但未可视化的设备对象，应在报告中列出。
2. 不得因为图元缺失而破坏整套拓扑出图流程。

### 10.3 自动布局失败

1. 若无法恢复清晰主干，应降级成分组块视图，而不是把所有对象堆成一行。
2. 降级图仍必须保持正交连线和基础可读性。

## 13. 验收标准

### 13.1 功能验收

1. 导入标准 `EQ/SSH/TP` 样例后，设备围绕拓扑分组展开，而不是按对象顺序平铺。
2. 连接线为标准正交路径，图纸主干走向清晰。
3. `Terminal` 不会在图纸上被误实例化为一次设备。
4. 含 `DL / DI` 的样例优先使用标准布局。
5. 缺少 `DL / DI` 时，仍能生成可读的自动布局图。

### 13.2 自动化验收

1. 拓扑分组结果可通过测试断言稳定复现。
2. 至少一组中等样例能验证“母线-开关-负荷/电源”展开方向。
3. 至少一组样例能验证 `DL / DI` 布局优先被采用。
4. 至少一组缺失拓扑或布局信息的样例能验证降级策略有效。

## 14. 结论

标准 CIM 导入后的正确目标，不是“把对象导进来就算完成”，而是把语义连接关系转成一张人能看懂的一次接线图。

这要求我们把 `TopologicalNode / ConnectivityNode` 作为连接核心，把 `Terminal` 留在语义和连接点层，把设备图元按拓扑方向展开，并在有 `DL / DI` 时优先复用标准布局。只有这样，CIM 导入场景才真正具备工程可用性。
